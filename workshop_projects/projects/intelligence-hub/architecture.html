Planner Agent

Analyzes document structure. Identifies type (textbook, paper, report), main topic, 5 key themes, target audience, and complexity level. Uses the cheap 8B model since this is a classification task.

Summarizer Agent

Writes 3-5 sentence executive summary grounded in source chunks. Extracts key points. Uses the Planner's themes to focus the summary. Every claim must come from the provided chunks.

Fact Extractor Agent

Extracts 6-10 important facts, numbers, formulas, and claims. Classifies each by type (fact/claim/formula/statistic) and importance (high/medium/low). Only extracts what's explicitly stated.

Quiz Generator Agent

Creates 5 MCQs testing understanding, not recall. Mix of 2 easy, 2 medium, 1 hard. Each question has 4 options, correct answer, and explanation. Tests concepts and application.

Gap Analyzer Agent

Identifies 3-5 missing or underexplained topics. Rates severity (critical/moderate/minor). Provides a coverage score (0-1) and recommendation. Helps identify areas for improvement.

Critic Agent (Quality Gate)

Validates ALL outputs against source chunks. Scores each section 0-1. If overall score < 0.7, triggers adaptive retry. Provides improvement hints that feed back to the Planner.

3. Parallel Execution & Error Boundaries

How 4 agents run concurrently with isolated failure handling

G--xTP: Exception caught! Note over TP: state["gaps"] = {} (empty) Note over TP: Error logged, pipeline continues end TP-->>M: Merge all results into main state Note over M: 3 succeeded, 1 failed = pipeline continues

State Copying

agent_state = {**state} creates a shallow copy for each agent. This prevents race conditions where agents overwrite each other's results.

Error Boundary Pattern

Each future.result() is wrapped in try/except. A crashed agent produces {} empty output. The Critic scores it as 0, but other agents' work is preserved.

Rate Limit Staggering

The threading.Lock() in call_llm ensures only one API call starts at a time with 4s delay. Parallel agents naturally stagger: t=0, t=4s, t=8s, t=12s.

Why Not Blind Retry?

With temperature=0, re-running the same prompt gives the same output. Adaptive retry changes the Planner's input by including the Critic's feedback, producing genuinely different analysis.

Budget Guard

tracker.check_budget() runs before each retry. If the budget can't cover another full cycle (~17K tokens), the pipeline stops gracefully instead of overspending.

Planner Agent

Reads the topic and document context. Defines fair FOR and AGAINST positions, identifies 3 debate dimensions. Uses call_llm_cheap (8B, 500 max tokens) since this is a structuring task.

Researcher FOR

No LLM call. Searches FAISS index with queries like "evidence supporting...", "benefits advantages...". Returns 8 evidence chunks sorted by relevance. Pure vector search = zero tokens.

Researcher AGAINST

Same as FOR researcher but with opposing queries: "evidence against...", "problems limitations...". Each side gets its own targeted evidence for stronger arguments.

Debater FOR

Builds 3 arguments with direct evidence quotes from chunks. Must include reasoning for each point and a preemptive rebuttal anticipating the AGAINST side's strongest argument.

Debater AGAINST

Same structure as FOR debater, arguing the opposite position. Uses its own evidence chunks. Both debaters work independently - they don't see each other's arguments until the Judge.

Judge Agent

Reads BOTH arguments. Scores each side on 5 criteria (0-10 each). Reports winner, margin, reasoning, and strongest/weakest points for each side. Impartial and evidence-based.

Cross-Examiner FOR

Only runs if Round 1 is close. Reads the AGAINST argument and finds its weakest point. Challenges it with evidence from FOR's chunks. Raises unanswerable questions.

Cross-Examiner AGAINST

Mirror of FOR cross-examiner. Reads FOR's argument and challenges its weakest point with AGAINST evidence. Both cross-examiners run in parallel.

Synthesizer Agent

Goes beyond "both sides have merit." Identifies specific common ground, names the core unresolved tension, and provides a nuanced conclusion. 150-200 words of balanced analysis.

3. Adversarial Parallel Execution

How FOR and AGAINST agents run simultaneously with isolated state

sequenceDiagram participant M as Main Thread participant TP as ThreadPool participant FOR as FOR Agent participant AGT as AGAINST Agent participant API as Groq API participant IDX as FAISS Index Note over M: Stage 4 - Research (Parallel) M->>TP: Submit researcher_for + researcher_against par FOR Research TP->>FOR: Start (state copy) FOR->>IDX: 5 semantic queries IDX-->>FOR: 8 evidence chunks FOR-->>TP: state.evidence_for = chunks and AGAINST Research TP->>AGT: Start (state copy) AGT->>IDX: 5 semantic queries IDX-->>AGT: 8 evidence chunks AGT-->>TP: state.evidence_against = chunks end TP-->>M: Merge: evidence_for + evidence_against Note over M: Stage 5 - Debate (Parallel) M->>TP: Submit debater_for + debater_against par FOR Debate TP->>FOR: Start (state copy + evidence_for) Note over FOR: Rate limit wait (5s) FOR->>API: call_llm_strong(evidence_for) API-->>FOR: 3 arguments + rebuttal FOR-->>TP: state.argument_for = JSON and AGAINST Debate TP->>AGT: Start (state copy + evidence_against) Note over AGT: Rate limit wait (5s) AGT->>API: call_llm_strong(evidence_against) API-->>AGT: 3 arguments + rebuttal AGT-->>TP: state.argument_against = JSON end TP-->>M: Merge: argument_for + argument_against Note over M: Both sides built independently - no information leakage

No Information Leakage

Each debater only sees its own evidence chunks. They build arguments independently. Neither side can see the other's position until the Judge reads both. This ensures a fair debate.

State Copying

s = {**state} gives each agent its own dict. After both finish, results are merged by side key: evidence_for, argument_for, evidence_against, argument_against.

Rate Limit Staggering

Even though agents run in parallel threads, the threading.Lock() ensures API calls happen sequentially with 5-second gaps. Parallel agents stagger: t=0s, t=5s.

Why Not Always Cross-Examine?

If the verdict is decisive (e.g., FOR: 42 vs AGAINST: 30), cross-examination won't change the outcome. Skipping saves 3 API calls (~8K tokens). Adaptive = efficient.

Cross-Examiners Read Opponents

These are the only agents that read another agent's output. Cross-Exam FOR reads state.argument_against and challenges its weakest point with evidence from state.evidence_for.

Verdict Can Flip

Round 2 can reverse the outcome. If FOR won Round 1 narrowly but AGAINST's cross-examination was devastating, the Judge may award Round 2 to AGAINST.
